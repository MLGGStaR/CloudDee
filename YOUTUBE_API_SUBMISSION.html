CloudDee — YouTube Data API v3 Quota Increase Submission

Application: CloudDee API Client: OAuth 2.0 Desktop client in Google Cloud project, named docket-cli API Services Used: YouTube Data API v3 (only) Source code: Private GitHub repository (server-to-server automation)

Summary

CloudDee is a single-user, server-to-server automation that publishes daily video breakdowns of newly released US public records (NTSB aviation and marine reports, Department of Justice press releases, SEC enforcement actions, federal court opinions). It runs as a GitHub Actions cron once a day and uses YouTube Data API v3 to upload original content the channel owner produced via an automated rendering pipeline.

There is no end-user interface, no public sign-up, and no third-party authentication. A single OAuth refresh token (the channel owner's) is stored as a GitHub Actions secret. The token is used exclusively to upload to the channel owner's own YouTube channel.

Architecture (per daily run)

┌──────────────────────────────────────────────────────────────────────┐
│  GitHub Actions cron (11:00 UTC daily)                               │
│                                                                      │
│  1. INGEST  Pull fresh public records from government sources        │
│             (NTSB CAROL, DOJ RSS, CourtListener, SEC EDGAR).         │
│             None of these are YouTube data.                          │
│                                                                      │
│  2. SCORE   Score records with Anthropic Claude (drama, novelty,     │
│             visual potential). None of this is YouTube data.         │
│                                                                      │
│  3. WRITE   Generate documentary script with Anthropic Claude.       │
│             Original commentary on public records.                   │
│                                                                      │
│  4. VOICE   OpenAI TTS narrates the script.                          │
│                                                                      │
│  5. RENDER  Fetch stock photos (Pexels) and generate visuals         │
│             (OpenAI image API). FFmpeg assembles final video.        │
│                                                                      │
│  6. UPLOAD  YouTube Data API v3 — see "API calls made" below.        │
└──────────────────────────────────────────────────────────────────────┘

The API client makes no read calls — no videos.list, no analytics, no search, no channel lookups, no viewer data access. The client is upload-only.

Authentication

  • A single OAuth 2.0 refresh token, obtained once via the standard InstalledAppFlow consent screen on the channel owner's local machine.
  • The refresh token is stored as a GitHub Actions encrypted secret (YT_REFRESH_TOKENS_JSON) and used by the cron worker to obtain short-lived access tokens.
  • Scopes used:
  • https://www.googleapis.com/auth/youtube.upload
  • https://www.googleapis.com/auth/youtube
  • No end-user authentication. No third party authenticates to this client.

Data handling

  • YouTube data accessed: none beyond the metadata immediately returned by videos.insert (the new video's id).
  • Storage: the returned video ID is logged to a SQLite file used to prevent re-uploading the same record (idempotency). It is never used for analytics, reporting, or commercialization.
  • Retention: less than 24 hours of operational logs; the SQLite file contains only the source-record-to-video-id mapping required for idempotency.
  • Sharing: no YouTube data is shared with anyone. There is no second party.

Commercialization

The CloudDee channel earns standard YouTube AdSense ad revenue once it qualifies for the YouTube Partner Program. The API client itself does not read viewer data, channel analytics, demographics, or any other YouTube- sourced signals — it only uploads original content.

Compliance

  • All facts in every video are sourced to the cited public-record document, whose URL is included verbatim in the video description.
  • Allegations against unconvicted parties are characterized as such ("alleged," "according to the indictment").
  • Sealed cases, identified minors, and sexual-assault matters are filtered out before script generation.
  • A standing correction policy is published in the description of every video.

Why we need an increase from the default 10,000-unit quota

At the planned cadence (2 long-form documentaries + 5 short-form videos per day) the daily upload load is approximately 12,200 quota units, slightly above the default 10,000. We are requesting an increase to 100,000 daily units to provide headroom for:

  • Steady-state operation at the planned volume.
  • Future expansion to additional source channels (ntsb_marine, courtlistener, etc., already supported by the codebase).
  • Re-uploads in the rare event a video must be replaced (e.g., for a factual correction).

Contact

The channel owner is the sole operator. All YouTube API operations are fully automated and traceable via the GitHub Actions run log.
